* 提交信息

    - 格式：

        第一行：提交主题

        第二行：空行

        第三行：包含提交原因的详细描述





    - 提交主题

        总结性的内容描述

        prefix 表情符号+提交内容总结+(issue 问题编号)



    - 规则：

        prefix

        表情符号

        英文命令形式记述

        50字以内

        结尾不要加句号

        提交内容的总结要写成自然语言



    - prefix

        在仓库中有多个应用程序的情况下，为了便于理解，需要明确标注进行更改的应用程序名称 [worker]



    - 表情符号

        🎨  :art:   UI显示修改

        🐎  :racehorse: 性能改进

        📝  :memo:  不涉及变更的文档或者文案修改

        🐛  :bug:   修改BUG

        ©️  :copyright: CI关联修改

        ✅   :white_check_mark:  追加测试

        🔒  :lock:  安全性相关的修改

        ⬆️  :arrow_up:  依赖关系升级

        ⬇️  :arrow_down:    依赖关系降级

        🚿  :shower:    不影响功能的代码风格修正（如空格、缩进等）或重构（不涉及功能变更但包含代码风格以外的更改）

        🆕  :new:   新增功能

        🔥  :fire:  删除功能

        🔧  :wrench:    功能更改

        🚋  :train: 基础设施代码的改进，配置文件的修正



    - 英文命令形式记述

        :memo: Update getting started documentation

        :fire: Remove deprecated methods

        :art: Refactor *** for readability



    - 问题编号

        :art: Change notification message (issue #3766)

        :art: Change notification message (idcf-private/issues#1234)



    - 第二行：空行

        由于提交主题与详细说明之间要用空行分隔，所以第2行留空。



    - 第三行：提交理由的详细说明

        写下进行该提交（更改）的理由。如果与Issue相关，写上Issue编号。
